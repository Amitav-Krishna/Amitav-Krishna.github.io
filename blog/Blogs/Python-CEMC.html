Exploring Number Criteria with Python

By Amitav Krishna

Introduction

I recently came across an interesting math problem. In it, you had to find out how many numbers fit a bunch of different criteria. I decided that it would be more fun to write a program in Python, so here we are. This problem was the Problem of the week from CEMC. Here is my program. The specifics of the problem are as follows:

Adrian has a few favourite numbers. In order to help people understand, he created a list of all the criteria:

  • Must have 5 digits
  • Must not have any even digits or repeated digits
  • 1 000s digit must be greater than 10 000s digit and 100s digit
  • 10s digit must be greater than 100s digit and unit digit

Approaching the problem

  • We know that this problem is asking us to go through every digit between 10 000 and 99 999, so that part should be simple. A for loop is what I’m using
  • There are 5 things we need to test for: No even digits, no repeated digits, the 1 000s digit must be greater than the adjacent digits, as well as the same thing for the 10s digit.
  • Finally, we just need to use a print() statement to output the answer.

The Setup

In programming, variables are essential for pretty much anything. You can imagine a variable as a box that holds a certain value. You can change the value inside the variable or you can use it for whatever purpose you want. For this program I had 2 variables outside of the for loop so that they didn’t reset for every number, favourites and numFavourites; favourites is a list that stores every one of Adrian’s favourite digits, while numFavourites stores the number of favourite numbers. Inside the loop I had 4 more variables: placeholder, we’ll get to its use later, digits, a list that will store all 5 digits of the number, numStr, the number converted to the string type (text), and isFavourite, a boolean (A true or false) that’s used to store whether or not a number is Adrian’s favourite.

Writing the for loop

If you’ve been coding for (Ha!) any amount of time, you should be familiar with for loops. Basically we have a variable, usually i, and then every time the loop repeats i increases by 1, until i reaches a certain value. This was the setup for my for loop: for i in numStr: In this for loop, it repeats for every digit inside numStr (The number we inputted). In each iteration of the loop, i takes on another one of numStr’s digits. This is what is inside the for loop:
digits.append(i)
if (int(digits[placeholder])) % 2 != 0 and isFavourite == True: isFavourite = True else: isFavourite = False placeholder += 1 o first digits.append(i) basically adds the value of i to the list. This is useful because it allows us to individually reference each one of the digits. The if statement is used to check if the digit is even. What (int(digits[placeholder])) does is it takes an index placeholder (Basically looks for that value inside of digits; for lists the first value is index 0) and then converts it to an integer. What the modulus operator (%) does is it takes the remainder of a number (For example: 14 % 3 = 2). We can use this to our advantage because if the remainder of a number divided by 2 is equal to 0, we know that this number is even. The rest of the if statement checks to see if isFavourite is true so that we don’t accidentally do the test again if it fails once. The else statement is so that if it fails the test, we set isFavourite to False so it can’t do any of the other tests. Finally we add 1 to placeholder so that when we iterate through the list again we do the next value.

Checking for repeating digits

In python there is something called a set. A set is different from a list because a set is unordered and cannot have duplicates. We can use this to our advantage by creating a set with all of the same digits as our list:
myset = set(digits)
Now we can compare the lengths of the set and the list in an if statement, and if it’s the same, we know there are no repeating digits:
if len(digits) == len(myset) and isFavourite == True:
isFavourite = True
else:
isFavourite = False
We have to again make sure that isFavourite is true so that we don’t accidentally give the number a chance to redeem itself.

Checking for the last 2 criteria

Programming the last two criteria are very simple, we just need to program a few inequalities:
if int(digits[1]) > int(digits[0]) and int(digits[1]) > int(digits[2]) and int(digits[3]) > int(digits[4]) and int(digits[3]) > int(digits[2]) and isFavourite == True:
isFavourite = True
else:
isFavourite = False
Not much to say about this one. We do again have to make sure that isFavourite is true, so that none sneak through.
